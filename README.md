# 구조 & 용어
# 오라클            vs  ES DB
# 테이블                인덱스
# 컬럼                  필드
# 스키마(테이블설계)     매핑

# 오라클:           vs ES DB           
# 언더바 표기법        인덱스(-표기법: 단어-단어)
# 대문자               필드(_표기법: 단어_단어)
#                      소문자

# 공식 용어(실무:금융,보험,기관) → 운영계-조회계 분리 아키텍처
# 업부, 기능별로 분리해서 운영하는 것
# 1) 오라클 DB      : 운영계, (트랜잭션, 데이터 무결성(신뢰성))
# 2) 엘라스틱서치 DB : 조회계, (빠른 검색(*), 통합 검색, 집계, 통계)
#    (과거~현재일부: 몽고DB 사용(빠른 검색, 일부 트랜잭션지원))
# 3) 레디스 DB       : 캐싱(데이터 결과 캐싱), 메모리 DB
# 아키텍처 패턴 관점 → Polyglot Persistence + CQRS
# Polyglot Persistence: 기능별로 분리 DB 운영하는 것
# CQRS               : 저장 + 조회 분리 DB 운영하는 것
